Create connectivity_extractor.py with a single public function:

def extract_connectivity(epochs, sid: str, cond: str) -> np.ndarray

Returns a 45-dim vector: 15 ROI pairs x 3 bands.

Implementation:

1. Cache check: path = CONN_CACHE_DIR / cond / f"{sid}.npz". If exists, return np.load(path)['conn'].

2. Enforce channel order: `epochs.reorder_channels(EEG_CHANNELS)` (or assert `epochs.ch_names == EEG_CHANNELS`) BEFORE computing connectivity. Without this, `np.triu_indices` ROI mapping will silently produce wrong results.

3. Compute wPLI via mne_connectivity.spectral_connectivity_epochs:
   - fmin/fmax tuples from CONN_BANDS values
   - method='wpli', faverage=True, verbose=False
   - Extract with `np.abs(con.get_data())` — wPLI returns values in [-1, 1]; we take absolute value since we care about magnitude of phase-lag index, not direction. Result shape: (325, 3)

4. ROI averaging:
   - Build ch_to_roi dict from ROI_GROUPS (channel -> region name)
   - row_idx, col_idx = np.triu_indices(len(EEG_CHANNELS), k=1) for 325 pairs
   - 15 ROI pairs via combinations_with_replacement(ROI_GROUPS.keys(), 2)
   - For each ROI pair: find matching channel-pair indices, average wpli. Flatten to (45,).

5. Cache save: mkdir parents, np.savez(path, conn=result).

Key details:
- Import from config: EEG_CHANNELS, ROI_GROUPS, CONN_BANDS, CONN_CACHE_DIR
- from mne_connectivity import spectral_connectivity_epochs
- from itertools import combinations_with_replacement
- Warn if len(epochs) < 10 but still compute
- For ROI pair matching: normalize order to match combinations_with_replacement canonical order (sorted tuple)
- 15 ROI pairs = 10 inter-ROI + 5 intra-ROI (combinations_with_replacement includes self-pairs, which is intentional — intra-ROI connectivity is informative)
  </action>
