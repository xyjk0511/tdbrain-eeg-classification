# Roadmap: TDBRAIN MDD vs ADHD 鉴别诊断

## Overview

从原始 TDBRAIN 数据到 MDD vs ADHD 分类验证的端到端 pipeline。四个阶段沿数据流推进：加载数据 -> 预处理 -> 提取特征 -> 分类验证并输出结果。每个阶段交付一个可独立验证的能力。

## Phases

- [x] **Phase 1: Data Loading** - 从 TDBRAIN 数据集加载 MDD/ADHD 受试者的静息态 EEG 数据 (completed 2026-02-23)
- [x] **Phase 2: Preprocessing** - 对原始 EEG 进行滤波、分段、伪迹剔除 (completed 2026-02-23)
- [x] **Phase 3: Feature Extraction** - 提取频谱特征（band power、TBR、FAA、相对功率） (completed 2026-02-23)
- [x] **Phase 4: Classification & Output** - 交叉验证分类、置换检验、结果输出 (completed 2026-02-23)

## Phase Details

### Phase 1: Data Loading
**Goal**: 用户可以从 TDBRAIN 数据集中按诊断筛选并加载 MDD/ADHD 受试者的 EEG 数据
**Depends on**: Nothing (first phase)
**Requirements**: DATA-01, DATA-02, DATA-03
**Plans**: 1 plan
Plans:
- [x] 01-01-PLAN.md — config + data_loader + main 验证脚本
**Success Criteria** (what must be TRUE):
  1. 运行脚本后能从 participants.tsv 中筛出 MDD 和 ADHD 受试者列表，数量与预期一致（MDD~320, ADHD~200）
  2. 能成功加载任意受试者的 .vhdr EEG 文件为 MNE Raw 对象
  3. 可通过配置参数选择 EC（闭眼）或 EO（睁眼）条件，加载对应文件

### Phase 2: Preprocessing
**Goal**: 原始 EEG 数据经过标准预处理后可用于特征提取
**Depends on**: Phase 1
**Requirements**: PREP-01, PREP-02, PREP-03
**Plans**: 1 plan
Plans:
- [x] 02-01-PLAN.md — preprocessor.py (filter + reref + epoch + artifact rejection) + main 验证
**Success Criteria** (what must be TRUE):
  1. EEG 数据经过 1-40 Hz 带通滤波和平均参考重参考
  2. 连续数据被切成固定长度 epoch（如 2s），可用于后续频谱分析
  3. 振幅超过 ±100 uV 的 epoch 被自动剔除，剔除比例可观察

### Phase 3: Feature Extraction
**Goal**: 从预处理后的 EEG 中提取 MDD/ADHD 鉴别相关的频谱特征矩阵
**Depends on**: Phase 2
**Requirements**: FEAT-01, FEAT-02, FEAT-03, FEAT-04
**Plans**: 1 plan
Plans:
- [x] 03-01-PLAN.md — feature_extractor.py (Welch PSD -> abs/rel band power + TBR + FAA) + main loop
**Success Criteria** (what must be TRUE):
  1. 每个受试者产出各频段（delta/theta/alpha/beta/gamma）的绝对功率特征
  2. 额叶通道的 Theta/Beta ratio 被正确计算（ADHD 核心指标）
  3. F3/F4 的 Frontal Alpha Asymmetry 被正确计算（MDD 核心指标）
  4. 各频段相对功率（频段功率/总功率）被计算并加入特征矩阵

### Phase 4: Classification & Output
**Goal**: 用交叉验证和置换检验验证频谱特征对 MDD vs ADHD 的分类能力，并输出可复现的结果
**Depends on**: Phase 3
**Requirements**: CLF-01, CLF-02, CLF-03, CLF-04, OUT-01, OUT-02
**Plans**: 1 plan
Plans:
- [x] 04-01-PLAN.md — classifier.py + main.py 扩展（SVM + 交叉验证 + 置换检验 + JSON 输出）
**Success Criteria** (what must be TRUE):
  1. 使用 StratifiedGroupKFold 按 subject 分组交叉验证，无数据泄露
  2. SVM(RBF) + 标准化 pipeline 完成训练和预测
  3. 控制台打印 AUC、准确率、敏感性、特异性和置换检验 p 值
  4. 1000 次置换检验完成，p 值报告在结果中
  5. 完整结果保存为 JSON 文件，包含所有指标和配置信息
